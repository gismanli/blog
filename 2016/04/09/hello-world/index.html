Begin:

平台:

  1. GitLab官网 (http://gitlab.org)
  2. 百度民间GitLab (http://gitlab.baidu.com)

前言:

之前更新版本一般有两种方法

  1. 采用写好的shell，使用crontab设定操作，隔段时间执行更新脚本，但是这种显然不是一种好的做法。
  2. 使用ssh压缩打包上传部署，例如使用Fabric，但公司的测试机是需要用堡垒机登录的，这种方法并不能实现自动化。

现在很多项目接入了gitlab, 我们就可以使用git push后自动执行更新脚本。

Git Hook

Git 会在某些特定事件触发hook，如果有需要，便可在相应位置插入需要的钩子，使用到的是 post-receive。
post-receive 是在 server 端，每次处理完 push 后触发。
