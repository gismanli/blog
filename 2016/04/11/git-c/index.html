GIT-C

Git auto check in, and get issue id from cooder.baidu.com automatic.

Usage

Commands

ci

check in and get issue from cooder

sync

Send to subversion by configs from .gitsync

说明

CHECK IN ./gitc ci

功能已独立，若无同步需求可忽略

同步 ./gitc sync

sync已经与ci完全独立，不再依赖issue_id，同步过程完全自动化，且无代码审批困扰

素材准备

创建同步用的GIT版本库目录

准备目标SVN，若不存在需手动创建，确保拥有写权限，否则会同步失败

在Git repo添加web hook

同步的目录使用的用户名和邮箱都是根据根目录的权限进行复制的，可以通过

本地目录 同步用的git版本库地址 目标svn版本库地址 Hi群ID
```

  • 以空格间隔，Hi群ID可以不设置（若要获取同步反馈请注意查收邮件）
  • “#” 开头为注释行
  • git可以使用http或ssh方式，ssh更加便捷

添加Hi群机器人

  • 添加习大大 >> linuxracer << 的Hi
  • 建一个Hi群，并在 famliy -> 个人中心 -> 我管理的Hi群组 中把该群转为外部群，防止添加的机器人被清掉
  • 把习大大添加到hi群即可，之后的同步就会收到同步反馈

开始同步

执行 ./gitc sync，在那看着提示就好了，然后习大大会发来贺电~
